# Автоматизация UI E2E тестирования проекта Toolshop Demo (Playwright)

Проект по автоматизации UI-тестирования современного веб-интерфейса интернет-магазина инструментов [**Toolshop Demo**](https://practicesoftwaretesting.com) с использованием **Python**, **Playwright** и паттерна **Page Object Model (POM)**.

---

## ⚙️ Стек технологий

* **Язык программирования:** Python 3.13.
* **Фреймворк тестирования:** Pytest
* **Инструмент автоматизации:** Playwright (Python)
* **Архитектурный паттерн:** Page Object Model (POM)
* **Отчетность:** Allure Report (с автоснятием скриншотов при падении)
* **CI/CD:** GitHub Actions (с автоматическим деплоем отчетов на GitHub Pages)

---

## 📁 Структура фреймворка

```text
├── .github/workflows/       # Конфигурация CI/CD пайплайна
│   └── run_tests.yml        # Автоматический запуск тестов и деплой Allure
├── pages/                   # Классы Page Objects (описание страниц и локаторов)
│   ├── __init__.py          # Маркер пакета Python
│   ├── login_page.py        # Страница входа (Sign In)
│   ├── catalog_page.py      # Каталог товаров, фильтры и корзина
│   └── checkout_page.py     # Многошаговое оформление заказа (Checkout)
├── tests/                   # Тестовые сценарии и конфигурация Pytest
│   ├── conftest.py          # Глобальные фикстуры, Faker, хук для скриншотов
│   └── test_e2e_purchase.py # Итоговый сквозной E2E-тест покупки
├── pytest.ini               # Системные настройки pytest и пути модулей
└── requirements
    ├── requirements.in      # Верхнеуровневые прямые зависимости проекта
    └── requirements.txt     # Скомпилированный бесконфликтный список зависимостей
```

## 🎯 Покрытые инженерные задачи и паттерны

В рамках проекта успешно реализован сквозной сценарий покупки (End-to-End) и решены ключевые проблемы UI-автоматизации:

1. **Обход динамических ID:** Проработаны капризные локаторы элементов с динамически генерируемыми UUID. Проблема решена через поиск по частичному совпадению CSS-атрибутов (`^=`) и фильтрацию элементов по стабильному тексту, что исключило ошибки строгого режима (Strict Mode Violation).
2. **Работа со сложными UI-компонентами:** Автоматизировано взаимодействие с динамическим DOM-деревом фреймворка Angular — управление чекбоксами боковой панели фильтров (метод `.check()`), проверка их состояния (`expect.to_be_checked()`) и нативная работа с выпадающими списками Dropdown (метод `.select_option()`).
3. **Data-Driven Testing (DDT) с Faker:** Исключены захардкоженные тестовые данные на этапе оформления заказа. Почтовые индексы, номера домов, улицы и штаты на этапе Billing Address генерируются изолированной фикстурой на лету.
4. **Эмуляция десктопного окружения и обход WAF (Stealth Mode в CI/CD):** Настроена глубокая конфигурация контекста Playwright для стабильного запуска в GitHub Actions. Задан принудительный десктопный размер виртуального холста (`1920x1080`), что полностью устранило проблему перехода сайта в мобильную верстку, а также реализована подмена `User-Agent` и заголовков локали для успешного прохождения защитных фильтров Cloudflare WAF.

## ⚠️ Специфика запуска в CI/CD (Обработка Cloudflare Turnstile)

Тестируемый веб-ресурс защищен системой **Cloudflare Turnstile**. При запуске тестов в облачной инфраструктуре **GitHub Actions** запросы исходят из пула IP-адресов дата-центров (Ubuntu/Azure), обладающих низкой сетевой репутацией для систем антифрода. В результате Cloudflare перехватывает сессию и вывешивает капчу-заглушку.

**Реализация защитных механизмов фреймворка:**
* Полный сквозной прогон (E2E) со всеми проверками каталога и чекаута успешно выполняется **локально** на хост-машине с "чистым" резидентным IP.
* Для работы в CI/CD в файл `conftest.py` добавлена автоматическая фикстура снятия скриншотов. Если в облаке GitHub Actions тест натыкается на капчу и останавливается, фреймворк мгновенно делает снимок экрана и прикрепляет его к отчету Allure. Это наглядно доказывает работоспособность системы фиксации ошибок при падении UI-тестов.

## 🛠️ Локальный запуск проекта

### Настройка окружения и компиляция зависимостей
```bash
python3.13 -m venv .venv
source .venv/bin/activate  # Для Linux/macOS
# или .venv\Scripts\activate для Windows

pip install pip-tools
pip-compile requirements/requirements.in
pip install -r requirements/requirements.txt
playwright install chromium
```

### Команды запуска тестов
* **Стандартный запуск в фоновом режиме (Headless):**
  ```bash
  pytest
  ```
* **Запуск с отображением браузера на весь экран:**
  ```bash
  pytest --headed
  ```
* **Запуск с замедлением действий для визуальной отладки:**
  ```bash
  pytest --headed --slowmo 500
  ```
* **Локальный сбор Allure-отчета:**
  ```bash
  pytest --alluredir=allure-results
  allure serve allure-results
  ```
